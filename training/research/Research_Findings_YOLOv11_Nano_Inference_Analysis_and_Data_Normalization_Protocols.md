# **Research Findings: YOLOv11 Nano Inference Analysis and Data Normalization Protocols**

Inference execution of the custom YOLOv11 Nano layout model on 24 evaluation images produced initial spatial detection data. Confidence scores for identified elements ranged from 0.25 to 0.72. The neural network successfully localized and classified discrete text blocks, including standard paragraphs and equations.

Specific detection failures occurred across multiple defined classes. The model failed to identify structural diagrams, specifically missing a beam cross-section, a spring assembly, and an RLC circuit. Headings remained unmapped. The cancellation class produced false positives, bounding valid mathematical calculations instead of isolated strikethroughs. Spatial accuracy degraded severely when processing source images containing a 90-degree rotation. Manual correction of these anomalous bounding boxes requires skewing adjustments that introduce spatial artifacts into the dataset. The absence of strict annotation parameters for intersecting spatial elements, such as lists overlapping with paragraphs or equations positioned adjacent to figures, contributes to classification errors.

The required solution for data normalization is hardware intervention. Physical flatbed scanners or dedicated document scanning software must process all source documents to provide a flat focal plane and eliminate perspective distortion prior to pipeline ingestion. Spatial intersection protocols require strict definition. Nested elements require hierarchical bounding boxes, while adjacent distinct elements require separate enclosures. The immediate operational step is packaging the 24 inference text files and source images into a single archive with the class definitions. This archive must be imported into the Label Studio environment. Manual correction of the generated coordinates and class assignments will establish the accurate ground truth data required for the subsequent model training iteration.
